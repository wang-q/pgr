# fa

`pgr fa` 模块是处理 **FASTA** 格式序列文件的瑞士军刀。它提供了从基础统计到复杂转换的一系列子命令，旨在替代常用的 shell 单行命令或 Perl/Python 脚本。

## 核心定位

- **定位**：FASTA 格式的全能操作工具箱。
- **输入**：FASTA 格式文件（支持 gzip/bgzip，支持 stdin）。
- **输出**：FASTA, TSV, 2bit 等格式。
- **互补**：
  - 上游：序列生成工具。
  - 下游：比对工具 (`lastz`, `minimap2`) 或分析工具。

## 子命令分组详解

### 1. 信息统计 (Info)
*快速了解序列文件的基本属性。*

- **`size`**: 计算每条序列的长度。
  - `--no-ns`: 仅计算有效碱基（排除 N）。
- **`count`**: 统计每条序列的碱基组成（A, C, G, T, N）。
  - 输出包含单条序列统计及文件总计。
- **`n50`**: 计算 N50, N90, 平均长度等组装统计指标。
  - `-N 50 -N 90`: 自定义 N 值。
  - `-g <size>`: 基于预估基因组大小计算 NG50。
  - `-S`/`-A`/`-E`/`-C`: 总长、平均长、E-size、序列数。
- **`masked`**: 识别被屏蔽（Masked）的区域。
  - 默认识别小写字母和 N。
  - `--gap`: 仅识别 N（Gap）区域。

### 2. 记录操作 (Records)
*提取、分割或重排序列记录。*

- **`one`**: 按名称提取单条序列。
  - 利用 `.fai` 索引（如果存在）加速。
- **`some`**: 根据名称列表提取多条序列。
  - `-i`: 反向选择（提取不在列表中的序列）。
- **`order`**: 按列表指定的顺序输出序列。
  - 适用于调整 FASTA 文件的记录顺序。
- **`split`**: 将大文件拆分为多个小文件。
  - `name`: 每条序列一个文件。
  - `about`: 按大小（字节数）拆分。支持 `--even` 保证记录数均匀。
- **`window`**: 滑动窗口切分序列。
  - `-l`/`-s`: 窗口长度和步长。
  - `--shuffle`: 随机打乱输出窗口（支持内存优化的分块打乱）。

### 3. 序列转换 (Transform)
*修改序列内容或属性。*

- **`rc`**: 反向互补（Reverse Complement）。
  - `-c`: 保持原名（不添加 `RC_` 前缀）。
  - 支持按列表仅处理部分序列。
- **`filter`**: 过滤和格式化序列。
  - 过滤：长度 (`--minsize`, `--maxsize`)、N含量 (`--maxn`)、去重 (`--uniq`)。
  - 格式化：转大写 (`--upper`)、IUPAC转N (`--iupac`)、去横杠 (`--dash`)、简化名称 (`--simplify`)、设置行宽 (`--line`)。
- **`dedup`**: 去除重复序列。
  - 模式：按名称（默认）、按序列内容 (`-s`)、按描述 (`-d`)。
  - `-b`: 同时比较反向互补链。
- **`replace`**: 根据 TSV 映射表重命名序列。
  - 支持一对多映射（序列复制）。
- **`mask`**: 根据区域文件（JSON）屏蔽序列。
  - 默认软屏蔽（转小写），`--hard` 硬屏蔽（转 N）。
- **`six-frame`**: 六框翻译。
  - 输出所有可能的 ORF。支持长度过滤、起始/终止密码子过滤。
- **`to-2bit`**: 转换为 2bit 格式。
  - 必需用于 `pgr chain` 等需要随机访问序列的工具。

### 4. 索引与随机访问 (Indexing)
*高效处理大文件的辅助工具。*

- **`gz`**: 使用 BGZF 格式压缩文件。
  - 生成 `.gz` 和 `.gzi` 索引。
  - 支持多线程并行压缩 (`-p`)。
  - BGZF 是随机访问压缩文件的基础（类似 `bgzip`）。
- **`range`**: 基于坐标提取子序列。
  - **必需**: 输入文件必须是 BGZF 格式（由 `pgr fa gz` 生成）。
  - 自动创建/使用 `.loc` 索引进行快速随机访问。
  - 格式：`chr1:1-100` 或 `chr1(-):100-200`（支持负链）。

## 典型用法

### 场景 A：数据清洗与统计
```bash
# 1. 过滤短序列并转为大写，统一行宽
pgr fa filter raw.fa --minsize 500 --upper --line 80 -o clean.fa

# 2. 统计 N50 和总长
pgr fa n50 clean.fa -N 50 -S
```

### 场景 B：提取特定区域
```bash
# 1. 压缩并建立索引
pgr fa gz genome.fa

# 2. 提取多个区域
pgr fa range genome.fa.gz "chr1:1000-2000" "chr2(-):500-1000" > regions.fa
```

### 场景 C：去重与切分
```bash
# 1. 按序列内容去重（考虑反向互补）
pgr fa dedup input.fa -s -b -o dedup.fa

# 2. 切分为约 10MB 大小的块用于并行处理
pgr fa split about dedup.fa -c 10000000 -o parts/
```
